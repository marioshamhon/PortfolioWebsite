ProJects

  • Inventory Management System: React

    This Inventory Management System is built with a modern, full‑stack architecture, utilizing React for a dynamic, user‑friendly frontend and an Express server for the backend logic, with data stored in a MongoDB database that I personally configured and optimized. The system provides robust capabilities for users to view, add, delete, and update inventory items. The React interface employs modern state management techniques—leveraging hooks like useState and useEffect—to create responsive forms and interactive views. On the server side, Express delivers a RESTful API that seamlessly communicates with MongoDB, ensuring that all changes in inventory data are processed efficiently. Overall, the application is designed for scalability and maintainability, offering a comprehensive solution for managing complex product data in real‑time.

  • Red Nose Pirate: C#

    This was a platformer game developed in a collaborative team environment using Unity and C#. Over a 4–6‑week sprint, I worked closely with my peers to design and implement engaging game levels and create enemy behaviors using state machines. I was responsible for much of the level design and for programming how enemy characters act and react within the game. Additionally, I actively participated in our GitHub version control workflow and conducted thorough code reviews to ensure robust code quality. Regular team meetings allowed me to contribute to key game design decisions, ensuring that our collective vision translated into an engaging player experience.

  • Linux Shell Clone: C

    Developed a fully functional clone of the Linux shell entirely in C, replicating key command-line functionalities. The project supports valid command execution, robust input/output redirection, effective pipeline commands, and the ability to run processes in the background. I implemented custom built‑in commands—including directory navigation and comprehensive error checking routines—to enhance usability and reliability. In addition, I leveraged low‑level system calls to manage process creation, inter‐process communication, and resource management, ensuring that the shell functions with proper error handling and efficiency. This project not only reinforced my understanding of Unix system internals and C programming but also demonstrated the ability to recreate essential operating system features from the ground up.
